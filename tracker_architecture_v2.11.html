Master architecture map

6 subsystems · all online

The tracker is one HTML file holding six subsystems in conversation. Data flows left-to-right; corrections flow right-to-left. Each subsystem owns a single responsibility: ingest reconciles three external sources; state derives one truth from the reconciled view; kpi, dop, and timeline each consume state and produce reports; the feedback loop reads what supervisors did with those reports and rebinds the OR-model constants accordingly.

v2.11 · key change The feedback loop became first-class in v2.11. Prior versions treated supervisor inputs as annotations on the rendered state. v2.11 routes them back through feedback.update(), which rebinds the OR-model constants and re-flows the state graph. The consequence: tonight's borrow event is in the κ table for tomorrow.

Data ingestion & reconciliation

Three sources arrive in different shapes and on different cadences. ingest.reconcile() normalizes them into a single ledger keyed by (scanner_id, sort_window, zone), then runs the identity-merge pass that resolves scanner-ID substitution events.

/ 03

State computation

The state module is the only place truth lives. Every downstream computation — KPI, DOP, Timeline — reads from state rather than from the raw ingested ledger. This is what makes the feedback loop work: corrections rebind state, not the data sources, so the next render reflects the correction without re-importing.

State carries: phase (Φ1 → Φ4), per-zone κ vector, current γ snapshot, scrubbed PPH ledger by scanner, active borrow/loan map. All other modules are pure functions of state.

/ 08

v2.11 fixes

8 corrections · all merged

v2.11 incorporates eight named corrections from the v2.10 audit cycle. Each was filed as a self-backtrack — the failure mode named before any code changed — and validated against the 613-sort corpus.

  • F1 · Feedback loop promoted to first-class subsystem. feedback.update() now rebinds κ live.
  • F2 · iGate-bounds assumption retired. Three sorts with >100% scan efficiency now treated as valid signal.
  • F3 · Scanner-ID substitution detection moved into reconcile pass (previously a separate audit).
  • F4 · Phase-engine boundary tightened — Φ4 wind-down explicitly excluded from PPH baseline.
  • F5 · LP-Lite degenerate case (PD5=214 solution) blocked at constraint construction.
  • F6 · Wilson exclusion logic corrected; previously over-excluded edge cases.
  • F7 · CURE OS=T filter applied at ingest, not at render.
  • F8 · Sort-window time guard: predictions outside 18:00–22:00 render a placeholder rather than extrapolating.
